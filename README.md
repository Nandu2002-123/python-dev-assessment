#Python Developer Assessment

## Project Overview
This repository contains a series of tasks completed as part of the Python Developer Assessment Program. The project demonstrates proficiency in Python fundamentals, version control with Git, code styling, data structures, OOP, error handling, and API interaction.


## Completed Tasks
- **Task 1.1:** Environment Setup & Hello World (`hello.py`)
## Goal - Python development environment ready and confirm basic functionality.

- **Task 1.2:** Git & GitHub Workflow setup.
## Goal - Understanding fundamental Git commands and manage your project on GitHub.

- **Task 1.3:** Linting and Formatting with Black/Flake8 (`bad_style.py`).
## Goal - Understanding the importance of consistent code style and use tools to enforce it.

- **Task 2.1:** Data Structures & Algorithms challenges (`dsa_challenges.py`).
## Goal is to demonstrate an understanding of fundamental Python data structures, such as lists and dictionaries, and the basics of algorithmic thinking.
## Implemented case-insensitive frequency mapping using Python dictionaries and used the sorted() built-in for efficient data organization.

- **Task 2.2:** OOP Implementation of a Book class (`book_store.py`).
## Goal is to demonstrate an understanding of classes, objectives, and methods in Python.

- **Task 2.3:** Debugging logical errors and Exception Handling (`debug_errors.py`).
## Goal is to learn how to anticipate and handle runtime errors gracefully and use a debugger to identify logical errors.
## try-except blocks to manage ZeroDivisionError and IndexError, ensuring the program remains resilient against edge-case inputs.

- **Task 3.1:** REST API interaction using the Requests library (`api_client.py`).
## Goal is to learn how to make HTTP requests to a REST API and process JSON responses using Python's requests library.
## Integrated the requests library to consume RESTful services, including robust error checking for network timeouts and non-200 HTTP responses.
